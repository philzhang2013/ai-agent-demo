## 1. 项目概述

### 1.1 项目简介

**Web Agent Demo** 是一个全栈 AI 智能助手应用，展示了流式输出、用户认证、工具调用和会话管理等核心功能。

**核心特性**：
- ✅ 实时流式对话（SSE）
- ✅ 用户认证系统（JWT）
- ✅ 多轮对话管理
- ✅ Function Calling（工具调用）
- ✅ ChatGPT 风格界面
- ✅ 完整的测试覆盖

### 1.2 学习路径

本项目是 **AI Agent 学习项目** 的第二阶段：

```
Demo 1: 命令行 Agent  →  Demo 2: Web 应用 Agent  →  Demo 3: 聊天机器人
   理解 Agent 核心循环        流式输出 + 前端集成           多轮对话 + 记忆
```

### 1.3 技术亮点

| 特性 | 实现方案 |
|------|----------|
| 流式输出 | SSE (Server-Sent Events) |
| 前端框架 | Vue 3 + TypeScript |
| 后端框架 | FastAPI (Python) |
| 数据库 | PostgreSQL + asyncpg |
| 认证方式 | JWT + bcrypt |
| API 文档 | Swagger UI (自动生成) |

## 4. 快速开始

### 4.1 环境要求

**必需软件**：
- Python 3.10+
- Node.js 18+
- PostgreSQL 14+
- git

**推荐工具**：
- VS Code + Python 插件
- Postman（API 测试）
- pgAdmin（数据库管理）

### 4.2 克隆项目

```bash
git clone https://github.com/philzhang2013/ai-agent-demo.git
cd ai-agent-demo/demos/02-web-agent
```

### 4.3 后端启动

```bash
cd backend

# 1. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 2. 安装依赖
pip install -e .

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入必要的配置

# 4. 初始化数据库
psql $DATABASE_URL < migrations/001_initial_schema.sql

# 5. 启动服务
uvicorn app.main:app --reload --port 8000
```

访问 http://localhost:8000/docs 查看 API 文档。

### 4.4 前端启动

```bash
cd frontend

# 1. 安装依赖
npm install

# 2. 配置代理（已配置，指向 localhost:8000）

# 3. 启动开发服务器
npm run dev
```

访问 http://localhost:5173 开始使用。

### 4.5 验证安装

```bash
# 后端健康检查
curl http://localhost:8000/api/health
# 预期输出: {"status":"ok","service":"web-agent-demo"}

# 前端访问
open http://localhost:5173
```

### 5.2 代码规范

#### 5.2.1 Python 代码规范

**命名约定**：
```python
# 函数和变量：小写+下划线
def get_user(user_id):
    pass

# 类名：大驼峰
class UserManager:
    pass

# 常量：大写+下划线
MAX_RETRIES = 3

# 私有方法：前缀下划线
def _internal_method(self):
    pass
```

**文档字符串**：
```python
async def process_message(message: str) -> AgentResponse:
    """
    处理用户消息并返回 AI 响应。

    Args:
        message: 用户消息内容

    Returns:
        AgentResponse: 包含响应内容和工具调用信息

    Raises:
        APIError: 当 API 调用失败时

    Example:
        >>> response = await agent.process_message("你好")
        >>> print(response.content)
        "你好！我是 AI 助手。"
    """
    pass
```

#### 5.2.2 TypeScript 代码规范

**命名约定**：
```typescript
// 接口：大驼峰 + I 前缀
interface IUser {
  name: string;
}

// 类型：大驼峰
type MessageRole = 'user' | 'assistant';

// 函数：小驼峰
function sendMessage() {}

// 常量：大写+下划线
const MAX_RETRIES = 3;
```

### 5.3 添加新功能

#### 5.3.1 添加新的 API 端点

**步骤**：

1. **创建路由文件**：`backend/app/api/myfeature.py`
```python
from fastapi import APIRouter, Depends

router = APIRouter(prefix="/api/myfeature", tags=["myfeature"])

@router.get("/")
async def get_items():
    return {"items": []}
```

2. **注册路由**：在 `backend/app/main.py` 中
```python
from app.api import myfeature
app.include_router(myfeature.router)
```

def test_should_get_items():

## 6. API 文档

### 6.1 认证 API

#### 注册用户
```http
POST /api/auth/register
Content-Type: application/json

{
  "username": "testuser",
  "password": "password123"
}
```

**响应**：
```json
{
  "message": "注册成功",
  "user": {
    "id": 1,
    "username": "testuser"
  }
}
```

#### 用户登录
```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "testuser",
  "password": "password123"
}
```

**响应**：
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "testuser"
  }
}
```

### 6.2 聊天 API

#### 非流式聊天
```http
POST /api/chat
Content-Type: application/json
Authorization: Bearer <token>

{
  "message": "你好",
  "session_id": "optional-session-id"
}
```

**响应**：
```json
{
  "response": "你好！我是 AI 助手。",
  "session_id": "session-123"
}
```

#### 流式聊天（SSE）
```http
POST /api/chat/stream
Content-Type: application/json

{
  "message": "你好"
}
```

**响应流**：
```
event: token
data: {"content": "你"}

event: token
data: {"content": "好"}

event: done
data: {"session_id": "session-123"}
```

## 9. 故障排查

### 9.1 常见问题

#### 问题 1：后端启动失败
```
Error: ModuleNotFoundError: No module named 'fastapi'
```

**解决方案**：
```bash
# 确保虚拟环境已激活
source venv/bin/activate

# 重新安装依赖
pip install -e .
```

#### 问题 2：数据库连接失败
```
Error: connection refused
```

**解决方案**：
```bash
# 检查 PostgreSQL 服务状态
sudo systemctl status postgresql

# 检查数据库连接字符串
echo $DATABASE_URL
```

#### 问题 3：API 返回 429 错误
```
{"error": "API 错误: 429"}
```

**解决方案**：
- 检查 API Key 余额
- 确认使用正确的端点（Coding Plan vs 标准端点）
- 减少请求频率

#### 问题 4：前端无法连接后端
```
Network Error
```

**解决方案**：
```bash
# 确认后端服务运行
curl http://localhost:8000/api/health

# 检查前端代理配置（vite.config.ts）
```
