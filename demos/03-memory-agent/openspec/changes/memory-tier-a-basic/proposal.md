# 长短记忆分层 - 阶段A（基础概念）

## 概述

实现AI助手的长短记忆分层机制，解决长对话上下文过长的问题。

## 背景

当前系统将所有历史消息发送给LLM，当对话轮数增加时：
- Token消耗过大
- 响应变慢
- 可能超出上下文限制

## 目标

实现基础的长短记忆分层：
1. **长记忆**：历史对话的摘要（压缩信息）
2. **短记忆**：最近N条原始消息（保持上下文连贯）

## 方案选择

**选择方案A：固定阈值触发**

- 当单会话消息数 > 10条时，自动触发摘要生成
- 简单易实现，适合A阶段学习
- 后续B阶段可优化为智能触发

## 预期行为

### 触发时机
```
用户发送消息 → 检查当前会话消息数
├── 消息数 >= 10 且未生成过摘要
│   → 调用LLM生成摘要
│   → 保存到 memory_summaries 表
│   → 后续请求：摘要 + 最近3条消息
│
└── 消息数 < 10 或已有摘要
    → 正常处理，发送全部消息
```

### API调用时上下文构建
```python
context = [
    {"role": "system", "content": "历史摘要: " + summary.content},  # 长记忆
    {"role": "user", "content": "..."},  # 最近消息
    {"role": "assistant", "content": "..."},
    ...
]
```

## 验收标准

- [ ] 新增 memory_summaries 表存储摘要
- [ ] 消息数达到10条时自动触发摘要生成
- [ ] 调用LLM生成会话摘要
- [ ] API调用时正确组合摘要+最近消息
- [ ] 测试覆盖率 >= 80%
- [ ] 所有测试先写后实现（TDD）

## 范围

### 包含
- 单会话内的记忆分层
- 自动摘要触发（固定阈值）
- 基础摘要生成

### 不包含（后续阶段）

- 向量检索（C阶段）
- 复杂压缩算法（B阶段）
