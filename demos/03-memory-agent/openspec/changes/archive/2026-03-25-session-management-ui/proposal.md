# Proposal: 会话管理 UI

## 概述

为 Demo 3 (03-memory-agent) 添加完整的会话管理 UI，包括会话列表侧边栏、会话切换、会话标题管理等功能。

## 问题陈述

当前 Demo 3 从 Demo 2 复制而来，虽然后端已有完整的会话管理 API，但前端缺乏相应的 UI：

- 用户无法看到历史会话列表
- 无法在不同会话之间切换
- 无法管理会话（删除、重命名）
- 新建会话需要手动调用 API

这限制了 Demo 作为"智能记忆 Agent"的展示效果，用户无法体验到多轮对话的持久化和切换能力。

## 解决方案

实现完整的会话管理 UI，包括：

### 核心功能

1. **会话列表侧边栏**：显示所有历史会话
2. **会话切换**：点击会话项切换到对应会话
3. **新建会话**：提供快速创建新会话的入口
4. **会话标题**：自动生成 + 手动编辑
5. **会话删除**：允许用户删除不需要的会话
6. **消息预览**：显示会话最后一条消息的预览

### 用户体验目标

- 侧边栏默认展开，便于快速切换
- 会话按更新时间倒序排列
- 当前会话高亮显示
- 响应式设计，支持小屏幕

## 成功标准

- 用户可以在侧边栏看到所有历史会话
- 用户可以点击切换到任意会话
- 用户可以创建新会话
- 用户可以删除会话
- 用户可以编辑会话标题
- 切换会话时，历史消息正确加载
- 前后端单元测试覆盖率 ≥ 80%

## 非目标 (Non-goals)

本阶段**不包括**：

- ❌ 跨会话的智能记忆（Phase 2）
- ❌ 对话摘要生成（Phase 3）
- ❌ 语义搜索功能（Phase 3）
- ❌ 会话分组或标签
- ❌ 会话导出功能
- ❌ 多设备同步

## 影响范围

### 前端新增

- `SessionSidebar.vue` - 会话列表侧边栏
- `SessionItem.vue` - 会话项组件
- `sessionStore.ts` - 会话状态管理

### 前端修改

- `HomeView.vue` - 添加侧边栏布局
- `ChatContainer.vue` - 集成会话切换
- `App.vue` - 调整布局容器

### 后端新增

- `migrations/002_add_session_title.sql` - 数据库迁移
- `PUT /api/sessions/{id}/title` - 更新会话标题 API

### 后端修改

- `app/db/repositories.py` - 扩展 SessionRepository
- `app/api/sessions.py` - 添加标题更新端点，扩展返回值

## 依赖关系

- 依赖 Demo 2 的现有代码基础
- 依赖后端已有的会话管理 API
- 无外部服务依赖

## 时间估算

- 数据库迁移：30 分钟
- 后端 API 扩展：1 小时
- 前端组件开发：3-4 小时
- 状态管理和集成：2 小时
- 单元测试：2 小时
- **总计：约 1 个工作日**

## 风险


| 风险          | 缓解措施             |
| ----------- | ---------------- |
| 会话切换时上下文加载慢 | 使用加载状态提示，优化数据库查询 |
| 标题自动生成逻辑不准确 | 支持手动编辑作为备选       |
| 侧边栏占用屏幕空间   | 响应式设计，支持折叠       |
